mybatis

MyBatis简介

  • 基于java的持久层框架

如何获得mybatis

  • maven
  • github

持久化

  • 方式：数据库（jdbc）,io文件持久化
  • 用处：有些数据很重要，为了防止断电失去

持久层

Dao层，Service层，Controller层…

  • 完成持久化工作的代码块
  • 层的界限十分明显

Mybatis用途

  • 传统JDBC代码很复杂，简化，框架
  • 方便
  • mybatis不是非必须的，更容易上手，技术没有高低之分

第一个MyBatis程序

思路：环境-导入Mybatis包-

  • 创建Maven项目
  • 在pom文件下导入mybatis包的和mysql包
  • 删除src自己创建一个maven模块
  • 在resource文件夹下创建mybatis-config.xml，写入相关代码（官网），连接mysql数据库
  • 创建MybatisUtils工具类
    1. 获取SqlSessionFactory对象
    2. 利用获得的SqlSessionFactory获得SqlSession连接（SqlSession 提供了在数据库执行 SQL 命令所需的所有方法）

个人遇到的问题：

  1. select语句要写具体的数据库select * from mybatis.user
  2. resource的配置文件无法识别，需要在pom文件中配置bulid

总结实体类pojo和工具类utils不需要发生变动

个人心得体会

  1. 创建maven工程十分方便，不需要手动导包，直接在pom文件下进行配置即可

  2. 在编写代码是注意结构的分层，比如创建Dao放置接口文件，pojo放置实体类（而实体类一旦创建后基本不需要发生改动），utils放置工具类，如MybatisUtil负责获得sqlSession（包含操作数据库的方法）

  3. mybaits中的三个概念

    1. SqlSessionFactoryBuilder负责创建SqlSessionFactory，创建完后基本不需要它了
    2. SqlSessionFactoryBuilder负责创建SqlSessionFactory
    3. SqlSessionFactory负责创建
      SqlSession（SqlSession session = sqlSessionFactory.openSession()）

  4. 几个文件的如何建立连接，如UserMapper(接口)和UserMapper.xml(实现接口)可以通过后者命名空间
    建立联系

  5. xml主要是通过空格以及“<”然后回提示一些方法，去使用即可，如实现接口的UserMapper中通过select标签

    id为重写的方法名称，resultType表示返回类型，返回值有基本类型,对象类型,List类型,Map类型等

个人总结

  1. 增删改查中增删改需要提交事务，查询不需要
  2. 实际编写过程为，编写UserMapper中的方法， UserMapper.xml编写sql语句 编写测试类
  3. 在测试类中有许多代码重复，如获取一个sqlSession对象，获得mapper接口，关闭sqlSession
    增删改需要提交事务（sqlSession.commit()）

分析错误

  1. 在namespace中绑定接口时使用.而不是/
  2. 标签不需要匹配错
  3. 在mybatis的配置文件中，绑定mapper需要使用路径
  4. 在idea中读错误需要从后往前读
  5. 配置文件需要符合规范
  6. Maven资源没有导出问题（添加build标签，配置资源过滤）
  7. 输出的xml文件中存在中文乱码问题，（可以在pom.xml的properties中添加）
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>

万能的Map

通过map可以随意自己设置参数，比如插入操作，不需要知道表中的特定字段就可以插入
当实体类或者数据库中的表，字段或者参数过多时，应该考虑使用Map

  • UserMapper接口
    int addUser2(Map<String,Object> map);

  • UserMapper.xml(注意参数名可以不用对应表中的字段)

  • 测试类

注意

  1. Map传递参数，直接在sql中取出key即可 parameterType="map"
  2. 对象传递参数，直接在sql中取对象的属性即可
  3. 只有一个基本类型参数，可以在sql中直接渠道
  4. 多个参数用Map,或者注解
